HTML obsoleto

Etiquetas de presentación obsoletas

  • <font> Para cambiar fuente, tamaño y color
  • <center> Para centrar contenido
  • <marquee> Texto que se desplaza (¡ese clásico de los 90s!)
  • <blink> Texto parpadeante
  • <big> y <small> (aunque <small> se recupero con un nuevo significado semántico)
  • <strike> Texto tachado, (ahora usa <del> o <s>)

Etiquetas de estructura obsoletas

  • <frame>, <frameset>, <noframes> Los frames fueron eliminados completamente en HTML5
  • <acronym> Reemplazado por <abbr>
  • <applet> Para Java applets (usa <object> o <embed>)

Tributos obsoletos

  • align, bgcolor, border en la mayoría de elementos
  • width y height en elementos que no sean <img>,<canvas>, etc.
  • cellpadding y cellspacing en tablas

Por qué quedaron obsoletos

La filosofía cambió hacia la separación de contenido y presentación. HTML debe describir la estructura y semántica del contenido, mientras que CSS maneja toda la presentación visual. Esto hace el código más mantenible, accesible y flexible.
